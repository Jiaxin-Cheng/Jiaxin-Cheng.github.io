本地仓库常用命令

命令 描述
git init 初始化仓库
git add 文件名称 将文件添加到暂存区
git commit -m 将文件提交到存储区并注释
git commit -a -m 将文件直接提交到存储区（前提是文件已经被跟踪）
git status 查看提交状态
rm -rf .git 删除仓库
命令 描述
git ls-files 查看存储区的文件
git log 查看提交的命令
git rm -f 文件名称 将文件从仓库中移除
git rm –cached 文件名 将文件从仓库里移除但保留暂存区
git log –pretty=oneline 将记录用一行显示
git reset –hard 命令id 回退到历史命令版本
git reflog 显示历史命令
git checkout –file 丢弃工作区的修改
git reset HEAD 文件名 将文件从暂存区移回工作区
命令 描述
git stash 隐藏当前工作区
git stash list 查看隐藏的
git stash pop 哈希码 恢复并删除隐藏的文件
git stash apply 哈希码 应用隐藏的文件
git stash drop 哈希码 删除隐藏的文件
git cherry-pick 哈希码 重复提交操作，用于当前版本的历史修改
git branch -D 分支名称 强行删除未合并的分支

分支管理

命令 描述
git branch 分支名称 创建分支
git checkout 分支名称 转换分支
git checkout -b 分支名称 创建分支并转换分支
git merge 分支名称 合并分支
git branch -d 分支名称 删除分支
git branch -a 查看所有分支
git switch -c 分支名称 切换分支
git merge –no-ff 分支名 普通模式合并

远程管理

配置远程仓库

远程仓库配置局部

命令 描述
git remote -v 查看远程库信息
git remote set-url origin 新仓库的URL
git push -u origin “分支名称” 第一次提交远程仓库时
git push origin 分支名 提交到远程
git remote add origin git@gitee.com :用户名/仓库名.git 关联远程仓库
git remote rm origin 删除本地库与远程库的关联
git push origin –delete 分支名称 删除分支，只是取消本地和远程的关联
git clone git@gitee.com:用户名/仓库名.git 将代码从远程库克隆下来

多人协作流程

  1. 先git push origin 分支名
  2. 如果有冲突git pull
    git branch –set-upstream-to 分支名 origin/分支名 本地与远程分支连接
  3. 在本地解决冲突再git push

打标签

命令 描述
git tag 标签名
git tag 标签名 命令id 给历史命令打标签
git tag 或者 git show 标签名 显示标签信息
git tag -d 标签名 删除标签
git push origin 标签名 远程推送标签
git push origin –tags 将标签全部远程推送
